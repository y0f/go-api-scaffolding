# 3. Generate type-safe DB and API code

Date: 2026-06-25

## Status

Accepted

## Context

The two most common sources of drift in a Go API are hand-written SQL that does
not match the schema, and API docs that do not match the handlers. ORMs hide SQL
and rely on runtime reflection; annotation-driven Swagger derives docs from code
comments that rot.

## Decision

The database layer is generated by sqlc from plain SQL against the real schema,
over pgx. The API types and server interface are generated by oapi-codegen from
`api/openapi.yaml`, which is the source of truth. Generated code lives in
`internal/gen`, is committed, and CI fails if it drifts from its sources.

## Consequences

Queries and the contract are checked at compile time, and the published OpenAPI
document always matches the server. The cost is a codegen step (`task generate`)
and the discipline of editing SQL and the spec rather than generated files.
